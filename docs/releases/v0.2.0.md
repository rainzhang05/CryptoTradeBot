# v0.2.0 Release Notes

## Summary

`v0.2.0` is the first release of the project as a rule-only Kraken spot bot operated through
the `cryptotradebot` command. This release folds in the full session change set:

- the rename from `tradebot` to `cryptotradebot`
- the interactive operator shell as the primary entrypoint
- the one-command `setup` workflow
- direct Kraken credential storage with `kraken auth set`
- removal of the ML, model-training, and research-sweep subsystem
- hardened deterministic strategy presets for `live_default` and `max_profit`
- corrected backtest accounting and more conservative profit reporting

This release keeps the V1 scope narrow:

- Kraken-only live execution
- USD-only accounting
- spot only
- deterministic rule-only strategy
- `simulate` and `live` runtime modes only

## Highlights

### New operator surface

- Bare `cryptotradebot` now launches the interactive operator shell on an interactive terminal.
- The default application home is now `~/.cryptotradebot/`.
- The public setup flow is now:
  - `cryptotradebot`
  - `setup`
  - `kraken auth set <api-key> --secret <api-secret>`
  - `run --mode simulate` or `run --mode live`

### One-command setup

`setup` now replaces the old split initialization and readiness flow. It:

- creates the operator home layout
- prepares runtime-ready canonical Kraken data
- completes and validates canonical data
- builds the deterministic feature cache needed for runtime
- checks configuration and Kraken public readiness
- reports Kraken private-auth readiness without echoing secrets

### Rule-only architecture

This release removes the ML stack completely. The project no longer includes:

- model training
- model validation or promotion
- active-model pointer handling
- prediction inference
- research sweeps tied to ML experiments

The bot is now fully deterministic and rule-only across backtest, simulate, and live mode.

### Strategy presets

Two named presets are available:

- `live_default`: hardened live/simulate default
- `max_profit`: aggressive upside-focused override

Both presets are rule-only and available through:

- `cryptotradebot backtest run --strategy-preset ...`
- `cryptotradebot run --mode simulate --strategy-preset ...`
- `cryptotradebot run --mode live --strategy-preset ...`

### Backtest accounting corrections

This release includes an accounting audit and fixes to the backtest profit-reporting path:

- the equity curve is now anchored from initial USD cash before the first executed trade
- yearly returns now use prior year-end equity as the base
- buy fees are now included in position cost basis so realized PnL is fee-aware on exit
- reports now include both mark-to-market ending equity and liquidation-adjusted ending equity
- reports now expose estimated terminal exit fee and slippage costs on remaining holdings

These changes make profit-rate reporting more conservative and more trustworthy without changing
the core strategy logic.

## Breaking Changes

- The root command is now `cryptotradebot`, not `tradebot`.
- The default application home is now `~/.cryptotradebot/`, not the previous tradebot home.
- `setup` replaces the old public bootstrap and doctor-style readiness flow.
- `kraken auth set` replaces manual credential setup as the documented operator path.
- The ML/model command surface and related artifacts are removed.
- The project is now explicitly rule-only in both implementation and documentation.

## Command Changes

### Added

- `cryptotradebot` interactive shell entrypoint
- `cryptotradebot setup`
- `cryptotradebot kraken auth set`
- `--strategy-preset` on runtime and backtest flows

### Removed

- public `doctor` operator command
- public bootstrap-data workflow
- model-training and model-promotion command paths
- ML experiment and sweep command paths

## Data and Runtime Notes

- Kraken remains the authoritative source for strategy decisions, backtests, simulation, and live
  evaluation.
- Binance and Coinbase remain supplementary validation and gap-repair sources only.
- The research default dataset track is `dynamic_universe_kraken_only`.
- Live and simulate mode still operate on daily point-in-time signals with continuous execution and
  freeze monitoring around that daily decision rhythm.

## Backtest Reporting Notes

Backtest reports now distinguish between:

- `total_return`: mark-to-market return at the last backtest timestamp
- `net_liquidation_total_return`: estimated return after liquidating all remaining open positions
  using the configured fee and slippage assumptions

Operators should treat `net_liquidation_total_return` as the more conservative endpoint when
comparing strategy outcomes.

## Upgrade Notes

Existing operators should migrate as follows:

1. Install or update the package so the `cryptotradebot` console script is available.
2. Move to the new application home or set `CRYPTOTRADEBOT_HOME` explicitly.
3. Run `cryptotradebot setup`.
4. Re-enter Kraken credentials with `cryptotradebot kraken auth set`.
5. Re-run backtests if you want reports with the corrected liquidation-adjusted accounting fields.

Legacy environment variable compatibility remains available for:

- `TRADEBOT_HOME`
- `BOT_CONFIG_PATH`

Preferred environment variables are now:

- `CRYPTOTRADEBOT_HOME`
- `CRYPTOTRADEBOT_CONFIG_PATH`

After upgrading, run:

```bash
cryptotradebot
setup

```

and confirm that the generated report includes:

- `strategy_preset`
- `dataset_track`
- `total_fees_usd`
- `net_liquidation_equity_usd`
- `net_liquidation_total_return`
